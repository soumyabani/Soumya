### Installation

‚Üí Install **Docker** and **Docker Compose**.

‚Üí Clone the CTFd repository:
```bash
git clone https://github.com/CTFd/CTFd.git
````

‚Üí Modify the `docker-compose.yml` file from the repository to specify a `SECRET_KEY` environment for the CTFd service.

1. Generate a `.ctfd_secret_key` file:

   ```bash
   head -c 64 /dev/urandom > .ctfd_secret_key
   ```
2. Export it as an environment variable:

   ```bash
   export SECRET_KEY=$(cat .ctfd_secret_key | base64)
   ```
3. Modify in `docker-compose.yml`:

   ```
   SECRET_KEY=${SECRET_KEY}
   ```

‚Üí Run:

```bash
docker compose up
```

‚Üí You should now be able to access CTFd at [http://localhost:8000](http://localhost:8000/)

> The default Docker Compose configuration files do not configure SSL/TLS.

---

### Dependencies

1. **WSGI server** - Installed by default - runs the web application - passes response for the requests from web server.
2. **Database server** - Default CTFd will create a SQLite database - used with MySQL, MariaDB, or SQLite - not recommended with PostgreSQL - CTFd supports JSON data type.
3. **Caching server** - The preferred caching server option is Redis.
